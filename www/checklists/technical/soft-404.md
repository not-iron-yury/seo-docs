# Чек-лист: Soft 404 и ошибочные страницы

<div class="text-block">
  <p>Настройка считается корректной, если:</p>

- все несуществующие URL возвращают 404 или 410
- страницы без контента не возвращают 200
- удаленные страницы не редиректятся на главную
- в индексе отсутствуют страницы без реальной SEO-ценности
</div>

<!-- --------- -->

## 1. Страницы без контента {#1}

<ul class='list-reset'>
  <li>1-1. Страницы без реального контента не возвращают <code>200</code></li>
  <li>1-2. Пустые категории не возвращают <code>200</code></li>
  <li>1-3. Пустые результаты поиска не возвращают <code>200</code></li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Если страница не содержит полезного контента, она должна возвращать <code>404</code> или <code>410</code>, либо быть закрыта от индексации.</p>
</div>

<!-- --------- -->

## 2. Пустые категории и фильтры {#2}

<ul class='list-reset'>
  <li>2-1. Категории без товаров не индексируются</li>
  <li>2-2. Фильтры без результатов не возвращают <code>200</code></li>
  <li>2-3. Пустые страницы не попадают в <code>sitemap</code></li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Пустые страницы должны либо возвращать <code>404</code>, либо быть закрыты от индексации.</p>
</div>

<!-- --------- -->

## 3. Страницы с сообщением об ошибке {#3}

<ul class='list-reset'>
  <li>3-1. Страница "товар не найден" не возвращает <code>200</code></li>
  <li>3-2. Страница "ничего не найдено" не возвращает <code>200</code></li>
  <li>3-3. Страница "контент удален" не возвращает <code>200</code></li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Если страница сообщает об отсутствии контента, HTTP-статус должен это отражать.</p>
</div>

<!-- --------- -->

## 4. Удаленные страницы {#4}

<ul class='list-reset'>
  <li>4-1. Удаленные страницы возвращают <code>404</code> или <code>410</code></li>
  <li>4-2. Удаленные страницы не возвращают <code>200</code> с текстом <i>"страница не найдена"</i></li>
  <li>4-3. Удаленные страницы не редиректятся на главную</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Удаленные URL должны возвращать корректный статус ошибки.</p>
</div>

<!-- --------- -->

## 5. Страница 404 {#5}

<ul class='list-reset'>
  <li>5-1. Возвращает статус <code>404</code></li>
  <li>5-2. Не возвращает <code>200</code></li>
  <li>5-3. Не индексируется</li>
  <li>5-4. Есть ссылки для навигации по сайту</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Страница ошибки должна корректно сигнализировать поисковым системам о несуществующем URL.</p>
</div>

<!-- --------- -->

## 6. Массовые soft 404 {#6}

<ul class='list-reset'>
  <li>6-1. Нет большого количества страниц с минимальным контентом</li>
  <li>6-2. Нет автогенерируемых страниц без ценности</li>
  <li>6-3. Нет тонких страниц (thin content), которые могут интерпретироваться как soft 404</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Страницы должны содержать такой объем контента, который ответит на поисковый интент.</p>
</div>

<!-- --------- -->

## Подводные камни {#hints}

- <strong>CMS возвращает 200 для страниц ошибок.</strong> Частая ошибка шаблонов.
- <strong>Удаленные товары в интернет-магазинах.</strong> Часто продолжают возвращать 200, но содержат сообщение "товар отсутствует".
- <strong>Пустые фильтры.</strong> Комбинации фильтров могут создавать тысячи пустых страниц.
- <strong>Редирект на главную вместо 404.</strong> Поисковые системы могут считать такие страницы soft 404.
- <strong>Тонкий контент.</strong> Страница может технически существовать, но быть распознана поисковой системой как soft 404.
