# Чек-лист: Микроразметка (Structured Data)

<div class="text-block">
  <p>Микроразметка настроена корректно, если:</p>

- тип Schema соответствует типу страницы
- разметка валидна и не содержит ошибок
- данные в разметке совпадают с контентом страницы
- страница может участвовать в rich results
- разметка присутствует на всех ключевых типах страниц сайта.

</div>

<!-- --------- -->

## 1. Наличие структурированных данных {#1}

<ul class='list-reset'>
  <li>1-1. На страницах используется микроразметка <code>Schema.org</code></li>
  <li>1-2. Разметка соответствует типу страницы (товар, статья, организация и т.д.)</li>
  <li>1-3. Микроразметка присутствует на ключевых типах страниц</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Каждая страница, имеющая значение для SEO, должна иметь структурированные данные, соответствующие её типу и содержимому.</p>
</div>

<!-- --------- -->

## 2. Корректность типов Schema.org {#2}

<ul class='list-reset'>
  <li>2-1. Используются релевантные типы разметки</li>
  <li>2-2. Тип разметки соответствует содержимому страницы</li>
  <li>2-3. Не используется несколько конфликтующих типов</li>
</ul>

<strong>Примеры:</strong>

| Тип страницы | Тип разметки          |
| ------------ | --------------------- |
| товар        | Product               |
| статья       | Article / BlogPosting |
| организация  | Organization          |
| контакты     | LocalBusiness         |
| навигация    | BreadcrumbList        |

<div class="text-block text-block--tip mt-30">
  <p>Тип разметки должен точно описывать сущность страницы.</p>
</div>
<!-- --------- -->

## 3. Формат микроразметки {#3}

<ul class='list-reset'>
  <li>3-1. Используется формат <code>JSON-LD</code></li>
  <li>3-2. Разметка размещена в <code>head</code> или <code>body</code> страницы</li>
  <li>3-3. Нет устаревших или смешанных форматов без необходимости<</li>
</ul>

<!-- --------- -->

## 4. Соответствие контенту {#4}

<ul class='list-reset'>
  <li>4-1. Значения в разметке совпадают с реальным контентом страницы</li>
  <li>4-1. В разметке нет данных, отсутствующих на странице</li>
  <li>4-1. Не используется скрытая или вводящая в заблуждение информация</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Все данные микроразметки должны присутствовать на странице и быть видимыми пользователю.</p>
</div>

<!-- --------- -->

## 5. Обязательные свойства {#5}

<ul class='list-reset'>
  <li>5-1. Указаны обязательные поля для конкретного типа <code>Schema</code></li>
  <li>5-2. Указаны основные атрибуты (<code>name</code>, <code>url</code>, <code>image</code> и т.д.)</li>
  <li>5-3. Нет пустых или некорректных значений</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Все обязательные свойства должны быть заполнены корректно.</p>
</div>

<!-- --------- -->

## 6. Ошибки и предупреждения {#6}

<ul class='list-reset'>
  <li>6-1. В разметке отсутствуют критические ошибки</li>
  <li>6-2. Нет предупреждений, влияющих на rich results</li>
  <li>6-3. Все свойства соответствуют спецификации <code>Schema.org</code></li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Микроразметка должна проходить валидацию без ошибок.</p>
</div>

<!-- --------- -->

## 7. Rich results {#7}

<ul class='list-reset'>
  <li>7-1. Разметка соответствует требованиям поисковых систем</li>
  <li>7-2. Страница может участвовать в расширенных результатах поиска</li>
  <li>7-3. Нет блокировки индексации страницы</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Страница должна иметь возможность получать расширенные сниппеты.</p>
</div>

<!-- --------- -->

## Подводные камни {#hints}

- <strong>Несоответствие контента.</strong> Разметка содержит данные, которых нет на странице.
- <strong>Чрезмерная разметка.</strong> На странице используется слишком много типов Schema.
- <strong>Ошибки обязательных полей.</strong> Некоторые свойства могут быть обязательными для rich results.
- <strong>Дублирование разметки. </strong> Несколько одинаковых блоков Schema на странице.
- <strong>JS-генерация разметки.</strong> Если микроразметка добавляется только после выполнения JavaScript, поисковый бот может её не увидеть.
- <strong>Неверный тип Schema.</strong> Например, использование Article на коммерческой странице.
- <strong>Спам-разметка.</strong> Использование рейтингов, отзывов или других данных, которых нет на странице.
