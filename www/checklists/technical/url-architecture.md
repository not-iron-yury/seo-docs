# Чек-лист: Архитектура URL

<div class="text-block">
  <p>Архитектура URL считается корректной, если:</p>

- URL отражает структуру сайта
- URL человекочитаемый
- каждая страница имеет одну каноническую версию URL
- отсутствуют технические и параметрические дубли
- структура URL стабильна и масштабируема

</div>

<!-- --------- -->

## 1. Логичность структуры URL {#1}

<ul class='list-reset'>
  <li>1-1. URL отражает структуру сайта</li>
  <li>1-2. URL соответствует иерархии разделов</li>
  <li>1-3. Структура URL понятна пользователю</li>
</ul>

```HTML
<!-- Корректная структура -->
/category/
/category/product/
/blog/article/
```

<div class="text-block text-block--tip mt-30">
  <p>URL должен отражать место страницы в структуре сайта.</p>
</div>

<!-- --------- -->

## 2. Читаемость URL {#2}

<ul class='list-reset'>
  <li>2-1. URL содержит человекочитаемые слова</li>
  <li>2-2. Нет случайных идентификаторов или хешей</li>
  <li>2-3. URL понятен без просмотра страницы</li>
</ul>

```HTML
/materials/chto-takoe-metategi/
/articles/prodvizhenie-sayta-statyami/
/category/bryuki-i-kombinezony-dlya-malchikov/
```

<div class="text-block text-block--info mt-30">
  <p>Для создания человекочитаемых URL используются сервисы транслитерации, плагины CMS, или собственные скрипты.</p>
</div>

<!-- --------- -->

## 3. Слова вместо параметров {#3}

<ul class='list-reset'>
  <li>3-1. Основные страницы не формируются через параметры</li>
  <li>3-2. Параметры не используются для ключевых страниц</li>
</ul>

```HTML
<!-- плохо -->
/category/?filter=muzhskie-shapki

<!-- хорошо -->
/muzhskie-shapki/
/muzhskie-shapki/?filter=sinie
```

<!-- --------- -->

## 4. Консистентность URL {#4}

<ul class='list-reset'>
  <li>4-1. Используется единый формат URL</li>
  <li>4-2. Нет смешивания разных структур</li>
  <li>4-3. URL формируются по единым правилам</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Все страницы одного типа должны иметь одинаковую структуру.</p>
</div>

<!-- --------- -->

## 5. Глубина URL {#5}

<ul class='list-reset'>
  <li>5-1. URL не содержит лишних уровней вложенности</li>
  <li>5-2. Структура URL не слишком длинная</li>
  <li>5-3. В URL нет технических сегментов</li>
</ul>

```HTML
<!-- плохо -->
/catalog/products/category/items/product-name/

<!-- хорошо -->
/catalog/category/product/

<!-- ещё проще -->
/category/product/
```

<div class="text-block text-block--tip mt-30">
  <p><strong>Глубина вложенности:</strong></p>

- важные страницы - 1-3
- категории - 1-2
- карточки товаров / статьи - 2-4
</div>

<!-- --------- -->

## 6. Разделители {#6}

<ul class='list-reset'>
  <li>6-1. В URL используются дефисы <code>-</code></li>
  <li>6-2. Не используются пробелы или специальные символы</li>
  <li>6-3. Не используются подчеркивания <code>_</code></li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Дефис - стандартный разделитель слов в URL.</p>
</div>

<!-- --------- -->

## 7. Статичность URL {#7}

<ul class='list-reset'>
  <li>7-1. URL не содержит сессионных параметров</li>
  <li>7-2. URL не меняется без необходимости></li>
  <li>7-3. Структура URL стабильна во времени</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>URL должен быть постоянным и не изменяться при обновлении контента.</p>
</div>

<!-- --------- -->

## Подводные камни {#hints}

- <strong>Генерация URL через CMS.</strong> CMS может создавать длинные и неконтролируемые URL.
- <strong>Фильтры и параметры.</strong> Могут генерировать огромное количество URL.
- <strong>Изменение структуры сайта.</strong> При миграциях URL могут измениться без корректных редиректов.
- <strong>Смешение языков в URL.</strong> Например английские и русские слова в одной структуре.
