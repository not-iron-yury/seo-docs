# Чек-лист: Sitemap.xml

<!-- --------- -->

<div class="text-block">
  <p> Карта сайта считается корректной, если:</p>

- содержит только индексируемые страницы со статусом 200
- включает канонические URL без параметров
- регулярно обновляется автоматически
- логически разделена для крупных сайтов
- полностью соответствует структуре сайта и внутренней перелинковке
</div>

<!-- --------- -->

## 1. Наличие и доступность {#1}

<ul class='list-reset'>
  <li>1-1. Файл доступен по стандартному пути: <code>/sitemap.xml</code></li>
  <li>1-2. Указан в robots.txt <code>(Sitemap: https://example.com/sitemap.xml)</code></li>
  <li>1-3. Отдается с кодом <code>200 OK</code></li>
  <li>1-4. Sitemap добавлен в панели вебмастеров</li>
  <li>1-5. Доступен без авторизации, IP-ограничений, JS</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Sitemap должен быть доступен для краулеров (без редиректов, промежуточных цепочек) и указан в robots.txt, чтобы поисковые системы могли быстро его обнаружить.</p>
</div>

<!-- --------- -->

## 2. Корректность структуры файла {#2}

<ul class='list-reset'>
  <li>2-1. Указаны версия и кодировка <code>version="1.0" encoding="UTF-8"</code></li>
  <li>2-2. URL указаны в абсолютном виде</li>
  <li>2-3. Нет синтаксических ошибок XML</li>
  <li>2-4. Нет битых символов, HTML-вставок</li>
  <li>2-5. Используется namespace (xmlns → <strong>обязательно</strong>)

```XML
<!-- Минимальный вариант -->
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">

```

  </li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Файл должен быть валидным XML и соответствовать официальному формату sitemap.</p>
</div>

## 3. Полнота и релевантность URL {#3}

<ul class='list-reset'>
  <li>3-1. Включены только канонические URL, предназначенные для индексации</li>
  <li>3-2. Нет страниц с noindex <code>noindex</code></li>
  <li>3-3. Нет страниц, закрытых в <code>robots.txt</code></li>
  <li>3-4. Нет <code>3xx / 4xx / 5xx</code> страниц</li>
  <li>3-5. Нет параметрических, фильтрационных и служебных URL (не имеющих SEO-ценности)</li>
  <li>3-6. Нет дубликатов (<code>http/https, www/non-www</code>)</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Sitemap содержит только индексируемые, приоритетные страницы, которые реально должны участвовать в ранжировании.</p>
</div>

<!-- --------- -->

## 4. Актуальность данных {#4}

<ul class='list-reset'>
  <li>4-1. Поле <code>lastmod</code> у каждой страницы соответствует реальному её изменению</li>
  <li>4-2. При массовом обновлении контента обновляются даты</li>
  <li>4-3. Нет одинаковых <code>lastmod</code> для всего сайта без причины</li>
  <li>4-4. Удаленные страницы отсутствуют в sitemap</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <ul>
    <li>Если в Sitemap появилась новая страница, то обновлять <code>lastmod</code> у всех страниц не нужно. Тег <code>lastmod</code> указывается для конкретной страницы и отражает дату её последнего изменения, а не дату обновления самого Sitemap.</li>
    <li>Тег <code>lastmod</code> отражает фактическое обновление контента, а не дату генерации файла.</li>
    <li>Массовая перегенерация дат приведет к тому, что поисковик начинает повторно краулить весь сайт без необходимости.</li>
  </ul>
</div>

<!-- --------- -->

## 5. Приоритеты и частота обновления {#5}

<ul class='list-reset'>
  <li>5-1. <code>priority</code> используется логично (главная > категории > карточки и т.д.)</li>
  <li>5-2. Нет одинакового <code>priority=1.0</code> для всех страниц (только для реально ключевых)</li>
  <li>5-3. <code>changefreq</code> не противоречит реальной частоте обновления</li>
  <li>5-4. Приоритеты отражают архитектуру сайта</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <ul>
    <li>Приоритет не влияет на ранжирование - это ориентир для краулеров.</li>
    <li>Если <code>changefreq</code> отсутствуют - поисковики определят частоту на основе своих данных.</li>
    <li>Главные страницы и ключевые разделы имеют более высокий приоритет, чем второстепенные страницы.</li>
  </ul>
</div>

<!-- --------- -->

## 6. Масштабирование и структура {#6}

<ul class='list-reset'>
  <li>6-1. В одном sitemap не более 50 000 URL</li>
  <li>6-2. Размер файла не превышает 50 MB (без gzip сжатия)</li>
  <li>6-3. Для большого сайта используется индекс-файл (<code>sitemap_index.xm</code>)</li>
  <li>6-4. Логично разделены типы страниц (товары, категории, статьи и т.д.)</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Система sitemap’ов должна быть структурированной, с индексным файлом для крупных проектов.</p>
</div>

<!-- --------- -->

## 7. Согласованность с реальной индексацией {#7}

<ul class='list-reset'>
  <li>7-1. Количество URL в <code>sitemap</code> сопоставимо с числом индексируемых страниц</li>
  <li>7-2. Практически все отправленные в Sitemap URL проиндексированы</li>
  <li>7-3. Нет URL, которые поисковики игнорируют из-за технических ошибок</li>
</ul>

## Подводные камни {#hints}

- <strong>Неактуальный sitemap.</strong> В файле могут оставаться удаленные страницы или старые URL.
- <strong>Несоответствие robots.txt</strong> Иногда в sitemap присутствуют страницы, закрытые от индексации.
- <strong>Редиректы в sitemap.</strong> URL могут вести на 301 или 302.
- <strong>Параметрические URL.</strong> В sitemap могут попадать страницы с фильтрами и параметрами без SEO-полезности.
- <strong>Автоматические генераторы sitemap.</strong> Некоторые CMS добавляют в карту сайта служебные страницы.
- <strong>Одинаковые приоритеты.</strong> Если все страницы имеют одинаковый priority, сигнал теряет смысл.
- <strong>Несоответствие canonical.</strong> В sitemap может быть указан неканонический URL.
