# Чек-лист: Canonical и дубли

<!-- --- -->

<div class="text-block">
  <p>Система canonical настроена корректно, если:</p>

- каждая индексируемая страница имеет self-canonical
- альтернативные версии URL указывают на одну основную страницу
- канонические URL возвращают 200
- в sitemap и внутренних ссылках используются только канонические URL
- отсутствуют индексируемые дубли страниц
</div>

<p class="mt-20">Почитать о canonical и дублях <a href="/docs/technical/canonical.html">в доке</a>.</p>

<!-- --------- -->

## 1. Наличие canonical {#1}

<ul class='list-reset'>
  <li>1-1. На всех индексируемых страницах присутствует <code>rel="canonical"</code><link></li>
  <li>1-2. Индексируемые страницы используют canonical, указывающий на собственный URL (self-canonical).</li>
  <li>1-3. Canonical присутствует в исходном HTML, а не добавляется через JS</li>
  <li>1-4. Используется абсолютный URL</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Каждая страница должна явно указывать канонический URL, который поисковые системы должны считать основной версией.</p>
</div>

<!-- --------- -->

## 2. Консистентность canonical {#2}

<ul class='list-reset'>
  <li>2-1. Canonical ведет на страницу со статусом 200</li>
  <li>2-2. Canonical не ведет на редирект</li>
  <li>2-3. Canonical не ведет на страницу с noindex</li>
  <li>2-4. Canonical не ведет на страницу, закрытую в robots.txt</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Каноническая страница должна быть доступна для индексации и возвращать статус <code>200</code>.</p>
</div>

<!-- --------- -->

## 3. Дубли URL {#3}

<p>Проверить, что canonical корректно решает следующие типы дублей:</p>

<ul class='list-reset'>
  <li>3-1. Версии <code>http</code> / <code>https</code></li>
  <li>3-2. Версии <code>www</code> / <code>non-www</code></li>
  <li>3-3. URL со слешем и без (<code>/page</code> и <code>/page/</code>)</li>
  <li>3-4. Параметрические URL (<code>?utm</code>, <code>?sort</code>, <code>?filter</code>)</li>
  <li>3-5. Пагинация и сортировка</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Все альтернативные версии URL должны указывать canonical на основную страницу.</p>
</div>

<!-- --------- -->

## 4. Параметры URL {#4}

<p>Проверить, что canonical корректно решает следующие типы дублей:</p>

<ul class='list-reset'>
  <li>4-1. Страницы с параметрами, которые не имеют SEO-ценности, имеют canonical на основную версию

```HTML
<!-- Такие страницы не имеют SEO-ценности -->
/category/?sort=price
/category/?view=grid
/category/?utm_source=ads
```

  </li>
  <li>4-2. Страницы с параметрами имеющие SEO-ценности должны иметь self-canonical

```HTML
<!-- Такие страницы полезны для SEO и могут индексироваться -->
/category/?filter=мужские-шапки
/category/?filter=настольные-лампы
```

  </li>
  <li>4-3. В sitemap присутствуют только канонические URL</li>
  <li>4-4. Внутренние ссылки ведут на канонические страницы</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Параметры не должны неконтролируемо создавать индексируемые дубли страниц.</p>
</div>

<!-- --------- -->

## 5. Пагинация {#5}

<p>Проверить, что canonical корректно решает следующие типы дублей:</p>

<ul class='list-reset'>
  <li>5-1. Пагинированные страницы не указывают canonical на первую страницу</li>
  <li>5-2. Canonical каждой страницы пагинации указывает на саму себя
  
  ```HTML
  <!-- Такие страницы должны иметь один свой собственный canonical -->
  /category/page/1
  /category/page/2
  /category/page/3
  ```
  </li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Каждая страница пагинации должна иметь self-canonical.</p>
</div>

<!-- --------- -->

## Рекомендуемая практика

| Тип URL                            | Canonical           |
| ---------------------------------- | ------------------- |
| фильтр по 1-2 параметрам с спросом | можно индексировать |
| фильтр по 2+ параметрам            | canonical / noindex |
| сортировка                         | на категорию        |
| UTM / служебные параметры          | на категорию        |
| пагинация                          | self-canonical      |
| языковые версии                    | self-canonical      |

### Практический ориентир

URL с параметром можно индексировать, если:

- есть поисковый спрос
- страница формирует отдельный кластер товаров
- у страницы есть уникальный title / h1 / текст
- количество таких страниц ограничено

Если этих условий нет - параметр лучше канонизировать на основную страницу.

## Подводные камни {#hints}

- <strong>Canonical может игнорироваться поисковиками.</strong> Если сигнал противоречит внутренним ссылкам или sitemap.
- <strong>Canonical на редирект.</strong> Частая ошибка после миграций сайта.
- <strong>Canonical на неканонический URL.</strong> Например на HTTP вместо HTTPS.
- <strong>Cross-domain canonical.</strong> Может случайно указывать на другой домен.
- <strong>CMS автоматически генерирует canonical.</strong> Иногда приводит к неправильным значениям.
- <strong>Canonical + noindex.</strong> Конфликтующие сигналы для поисковых систем.
- <strong>Дубли в параметрических URL.</strong> Особенно часто встречается на e-commerce сайтах с фильтрами.
