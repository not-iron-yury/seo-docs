# Чек-лист: Mobile-first рендеринг и адаптивность

<div class="text-block">
  <p>Mobile-first оптимизация считается корректной, если: </p>

- мобильная версия содержит весь основной контент страницы
- структура внутренних ссылок доступна и индексируема
- страница полностью рендерится без необходимости выполнения сложного JavaScript
- интерфейс корректно отображается на основных разрешениях мобильных экранов

</div>

<!-- --------- -->

## 1. Mobile-first версия страницы {#1}

<ul class='list-reset'>
  <li>1-1. Контент для всех версий сайта (mobile, desctop) - одинаковый</li>
  <li>1-2. На мобильной версии присутствуют все ключевые SEO-элементы (текст, заголовки, ссылки)</li>
  <li>1-3. На мобильной версии присутствуют микроразметка</li>
  <li>1-4. Canonical, meta robots и другие технические теги одинаковы для всех версий</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Мобильная версия должна содержать тот же основной контент и SEO-сигналы, что и десктопная.</p>
</div>

<!-- --------- -->

## 2. Корректность адаптивной верстки {#2}

<ul class='list-reset'>
  <li>2-1. Используется адаптивная верстка (responsive design)</li>
  <li>2-2. Присутствует meta-тег viewport</li>
  <li>2-3. Контент корректно масштабируется на разных ширинах экранов</li>
  <li>2-4. Нет горизонтального скролла</li>
  <li>2-5. Нет обрезанных элементов интерфейса</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Страница должна корректно отображаться на экранах разных размеров без необходимости масштабирования.</p>
</div>

<!-- --------- -->

## 3. Доступность контента для поисковых систем {#3}

<ul class='list-reset'>
  <li>3-1. Контент не скрывается на мобильной версии через CSS/JS</li>
  <li>3-2. Нет «облегченной» мобильной версии без текста</li>
  <li>3-3. Lazy-loading не скрывает основной контент от краулеров</li>
  <li>3-4. Вкладки, аккордеоны и скрытые блоки индексируемы</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Контент должен присутствовать в DOM и быть доступным для парсинга поисковым ботом.</p>
</div>

<!-- --------- -->

## 4. Навигация и внутренние ссылки {#4}

<ul class='list-reset'>
  <li>4-1. Мобильное меню содержит основные разделы сайта</li>
  <li>4-2. Внутренняя перелинковка доступна на мобильной версии</li>
  <li>4-3. Нет скрытых важных ссылок, присутствующих только на десктопе</li>
  <li>4-4. Хлебные крошки отображаются корректно в обеих версиях</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Структура ссылок должна быть одинаково доступна для мобильных пользователей и поисковых систем.</p>
</div>

<!-- --------- -->

## 5. UI-элементы и удобство взаимодействия {#5}

<ul class='list-reset'>
  <li>5-1. Кнопки и ссылки имеют достаточный размер для клика</li>
  <li>5-2. Нет элементов, перекрывающих контент</li>
  <li>5-3. Попапы и interstitial-блоки не мешают просмотру страницы</li>
  <li>5-4. Формы и фильтры удобны для мобильного ввода</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Интерфейс должен быть удобен для взаимодействия на сенсорных устройствах.</p>
</div>

<!-- --------- -->

## 6. Проверка ключевых шаблонов страниц {#6}

<ul class='list-reset'>
  <li>6-1. Главная страница</li>
  <li>6-2. Категории</li>
  <li>6-3. Карточки товаров / услуг</li>
  <li>6-4. Контентные страницы</li>

<div class="text-block text-block--tip mt-30">

</div>

## Подводные камни {#hints}

- <strong>Разный контент на mobile и desktop.</strong> В мобильной версии иногда скрывается часть текста или блоков.
- <strong>JS-рендеринг интерфейса.</strong> Меню, фильтры и вкладки могут появляться только после выполнения JavaScript.
- <strong>Hidden-content через CSS.</strong> Контент присутствует в коде, но скрыт через display:none.
- <strong>Гамбургер-меню без внутренних ссылок.</strong> В некоторых реализациях важные разделы бывают доступны только на десктопе.
- <strong>Интерстициальная реклама.</strong> Попапы могут перекрывать контент и ухудшать пользовательский опыт.
- <strong>Lazy-loading контента.</strong> Неправильная реализация может препятствовать индексации изображений и текста.

Динамические интерфейсы e-commerce
Фильтры и сортировки могут не работать корректно на мобильных устройствах.
