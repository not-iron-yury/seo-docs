# Чек-лист: HTTPS и безопасность

<div class="text-block">
  <p>Корректная HTTPS-конфигурация: </p>

- HTTP → 301 → HTTPS
- не смешиваются HTTP и HTTPS
- все внутренние ссылки используют HTTPS
- canonical указывает на HTTPS
- 100% страниц доступны по HTTPS
- 1 каноническая версия сайта

</div>

<!-- --------- -->

## 1. Наличие на всех страницах {#1}

<ul class='list-reset'>
  <li>1-1. Доступность сайта по протоколу <code>HTTPS</code></li>
  <li>1-2. HTTP-версия не используется как основная</li>
  <li>1-3. Редирект HTTP-страниц на HTTPS работает для всех страниц, а не только для главной.</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>HTTPS является официальным фактором ранжирования и стандартом безопасности.</p>
</div>

<!-- --------- -->

## 2. Смешанный контент {#2}

<ul class='list-reset'>
  <li>2-1. На HTTPS-страницах все ресурсы загружаются через <code>HTTPS</code></li>
  <li>2-2. Список ресурсов к проверке:
    <ul class="ml-30">
      <li>изображения</li>
      <li>скрипты</li>
      <li>CSS</li>
      <li>iframe</li>
      <li>шрифты</li>
    </ul>
  </li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Смешанный контент вызывает предупреждения браузера.</p>
</div>

<!-- --------- -->

## 3. Индексация {#3}

<ul class='list-reset'>
  <li>3-1. В sitemap только <code>HTTPS URL</code></li>
  <li>3-3. Canonical указывает на <code>HTTPS URL</code></li>
  <li>3-4. Внутренние ссылки используют <code>HTTPS</code></li>
  <li>3-2. Адрес <code>sitemap</code> должен использовать <code>HTTPS</code></li>
</ul>

```HTML
Sitemap: https://site.ru/sitemap.xml
```

<!-- --------- -->

## Подводные камни {#hints}

- <strong>Частичный редирект.</strong> Иногда редирект настроен только для главной страницы.
- <strong>Старые шаблоны.</strong> После перехода на <code>HTTPS</code>, в старых статьях и старых шаблонах CMS часто остаются ссылки с <code>HTTP</code>.
- <strong>Внешние HTTP-ресурсы.</strong> Иногда сайт подключает внешние ресурсы (шрифты, скрипты, виджеты). Это тоже приводит к смешанному контенту.
- <strong>Истекший сертификат.</strong> Браузеры показывают предупреждение безопасности, резко снижает доверие пользователей.
- <strong>Неправильный canonical.</strong> Иногда страница открывается по <code>HTTPS</code>, а её <code>rel="canonical"</code> указывает на <code>HTTP</code>.
