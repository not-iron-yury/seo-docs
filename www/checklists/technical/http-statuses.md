# Чек-лист: Проверка статусов HTTP

<!-- --------- -->

## 1. Общая проверка статусов {#1}

<ul class='list-reset'>
  <li>1-1. Все страницы сайта возвращают корректный HTTP-код:
    <ul class="ml-30 ">
      <li><code>200</code> - для индексируемого контента</li>
      <li><code>301</code> - для постоянных перенаправлений</li>
      <li><code>302</code> - для временных перенаправлений</li>
      <li><code>404</code> - для несуществующих страниц</li>
      <li><code>410</code> - для удаленных страниц (по желанию для ускорения удаления из индекса)</li>
    </ul>
  </li>
  <li>1-2. Нет <code>500</code>  / <code>503</code> / <code>504</code> страниц для пользователей и поисковиков</li>
  <li>1-3. Проверка c <code>www/non-www</code>, с <code>http/https</code> редиректами</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Каждый URL должен иметь логичный код, соответствующий его статусу и роли в индексации.</p>
</div>

<!-- --------- -->

## 2. Редиректы {#2}

<ul class='list-reset'>
  <li>2-1. Постоянные изменения используют <code>301</code>, а не <code>302</code></li>
  <li>2-2. Цепочки редиректов ≤2 (лучше 1)</li>
  <li>2-3. Нет редирект-петель и циклов</li>
  <li>2-4. Редиректы ведут на конечную, каноническую версию страницы</li>
  <li>2-5. Тестируются редиректы с <code>www/non-www</code>, <code>http/https</code>, разных вариаций URL</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Минимальные цепочки, без дублирования, редирект ведет на правильный URL.</p>
</div>

<!-- --------- -->

## 3. 4xx и 410 страницы {#3}

<ul class='list-reset'>
  <li>3-1. Ошибочные URL отдают 404/410, а не 200 с контентом «Страница не найдена»</li>
  <li>3-2. Страница с 410 используется для удаленного контента, если нужно ускорить исключение из индекса</li>
  <li>3-3. Нет «мягких 404» - когда страница возвращает 200, но фактически пустая или с сообщением об ошибке</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Soft 404 нужно исправлять: либо реальный 404/410, либо релевантный контент.</p>
</div>

<!-- --------- -->

## 4. Ошибки 5xx {#4}

<ul class='list-reset'>
  <li>4-1. Нет временных или постоянных серверных ошибок</li>
  <li>4-2. Проверяется в пиковые нагрузки и при сканировании поисковиков</li>
  <li>4-3. Логируются для оперативного устранения</li>
</ul>

<div class="text-block text-block--tip mt-30">
  <p>Сайт всегда должен отдавать стабильные ответы для пользователей и краулеров.</p>
</div>

<!-- --------- -->

## Подводные камни {#hints}

- Цепочки редиректов - длинные редиректы тормозят краулинг, теряется вес ссылок.
- Разные версии протоколов - http и https могут давать разные статусы.
- Ошибки серверов под нагрузкой - иногда появляются 5xx только при пиковом трафике.
- Неучтенные внутренние ссылки - URL в контенте, sitemap или меню может отдавать другой статус.
- Динамические страницы с параметрами - параметры могут возвращать разные коды, создавая ложные сигналы.
