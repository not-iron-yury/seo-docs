# Дубли (контент и URL) {#duplicates}

<ul class="info-block">
  <li class="info-block-itm">Задача: Проверить системную генерацию дублей</li>
  <li class="info-block-itm">Важность: Очень высокая</li>
</ul>

## Когда проверять {#when}

- Перед релизом сайта (раздела) и повторно сразу после релиза.
- После изменений в логике URL (редизайн, новая фильтрация, пагинация, роутинг, SSR/CSR логика).
- При падении SEO-трафика (возможно каннибализация и дубли).
- Регулярно в процессе (раз в 1–3 месяца) - отлавливать накопившиеся дубли и SEO-деградацию.

## Что проверять {#what-to-check}

- Content → Duplicates
- URL

## Должно быть {#must-be}

- Уникальность ключевых страниц
- Дубли только там, где это логически оправдано

## Источник проблем {#problems}

<ol>
  <li>Параметрические URL
  
  
```
/products
/products?utm_source=google
/products?ref=affiliate
/products?session_id=123
```
URL с query-параметрами (?), которые создают дубли одного и того же контента. По сути это одна страница, но для поисковика это разные URL, из-за чего размывается вес, и возможна каннибализация.

  </li>
  <li>Сортировки

```
/products?sort=price_asc
/products?sort=price_desc
/products?sort=popular
```

Разные варианты сортировки генерируют страницы с одинаковым набором товаров, но в другом порядке. Контент почти идентичен и поисковик видит дубли с минимальными отличиями.

  </li>
  <li>Пагинация
  
```
/blog?page=1
/blog?page=2
/blog?page=3
```
Проблема если есть частично повторяющиеся элементы, дубли мета-тегов title и description, у страниц пагинации тонкий (thin) контент.

  </li>
</ol>

## Что делать {#to-do}

- Правильная расстановка `canonical`
- Закрытие от индексации через `noindex`
- нормализация URL
