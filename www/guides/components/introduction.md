# SEO-friendly компоненты {#seo-friendly-components}

## Базовый принцип {#1}

<div class="text-block text-block--info">
  <p>Контент всех вкладок должен присутствовать в HTML страницы при её загрузке.</p>
  <p>JavaScript должен только переключать отображение вкладок, а не создавать их содержимое.</p>
</div>

Почти для всех компонентов действует один и тот же базовый принцип:

- 1. Поисковик должен получить весь контент в исходном HTML (SSR или prerender):
  <ul class="ml-30">
    <li>присутствующий в DOM сразу,</li>
    <li>с корректной семантикой,</li>
    <li>без JS как “единственного способа существования”.</li>
  </ul>

- 2. Навигация должна использовать `<a href>`.
- 3. Состояния интерфейса должны иметь понятную URL-логику (или быть закрытым от индексации).
- 4. JavaScript это только улучшение UX, не источник контента.

Отсюда вытекают все решения.

## Что на самом деле проверяет поисковик {#2}

Поисковик НЕ проверяет:

- “видно ли пользователю”
- “активна ли вкладка”

Поисковик проверяет:

- наличие текста в DOM
- логично ли он встроен в структуру документа
- не выглядит ли это как попытка обмана

## CSS-сокрытие {#3}

CSS:

- не удаляет узел,
- не меняет структуру документа,
- не ломает индексацию.

JS:

- лишь переключает классы / атрибуты,
- не влияет на факт существования текста.

  <p>Поисковик видит длинную страницу с логическими секциями, часть из которых свёрнута интерфейсом.</p>
</div>
