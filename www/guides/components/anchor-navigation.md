# Якорная навигация внутри страницы {#anchor-navigation}

<ul class="info-block">
  <li class="info-block-itm">Тип компонента: Внутренняя навигация страницы</li>
  <li class="info-block-itm">SEO-риск: Низкий </li>
  <li class="info-block-itm">Критично для: Crawlability и UX длинных страниц</li>
</ul>

Якорная навигация используется на страницах с большим объёмом контента и позволяет пользователю быстро переходить к нужному разделу страницы.

## Суть проблемы {#point}

Реализация якорной навигации через JavaScript-обработчики вместо обычных ссылок приводит к ряду проблем:

- ссылки не воспринимаются поисковыми системами как навигационные
- ухудшается crawlability (индексационная доступность) страницы
- может нарушаться доступность интерфейса

## Правильная модель {#correct-model}

- Якорная навигация должна использовать обычные ссылки `<a href="#section-id">`, которые ведут к элементам страницы с соответствующим id.
- Она должна работать без JavaScript, как стандартный HTML-механизм перехода к элементам страницы.
- JavaScript может использоваться только для дополнительных эффектов (например, плавная прокрутка).

## Пример {#examples}

::: code-group

```HTML

<!-- якорная навигация -->
<nav class="page-nav">

  <a href="#description">Описание</a>
  <a href="#features">Характеристики</a>
  <a href="#reviews">Отзывы</a>

</nav>

<!-- секции страницы -->
<section id="description">
  <h2>Описание</h2>
  <p>Описание услуги...</p>
</section>

<section id="features">
  <h2>Характеристики</h2>
  <p>Характеристики продукта...</p>
</section>

<section id="reviews">
  <h2>Отзывы</h2>
  <p>Отзывы клиентов...</p>
</section>
```

```CSS

/* плавная прокрутка */
html {
  scroll-behavior: smooth;
}
```

:::

## Как не нужно {#how-not-to}

<ul class="list-reset">
  <li>❌ Использовать <code>onclick</code> вместо ссылок</li>
  <li>❌ Реализовывать переход через <code>scrollTo()</code> без <code>a href</code></li>
  <li>❌ Генерировать id элементов <strong>динамически</strong> через JS</li>
  <li>❌ Создавать якорную навигацию, которая работает только после загрузки JavaScript</li>
  <li>❌ Использовать <code>button</code> вместо <code>a href</code></li>
</ul>

## Чек-лист {#checklist}

<ul class="list-reset">
  <li>✔️ Навигация использует <code>a href="#section"</code></li>
  <li>✔️ У каждой секции страницы есть уникальный <code>id</code></li>
  <li>✔️ Якорные ссылки работают без JavaScript</li>
  <li>✔️ Переход по ссылке изменяет URL-фрагмент <code>#section</code></li>
  <li>✔️ Ссылки доступны в HTML исходнике страницы</li>
</ul>

## Инструменты проверки {#checking-tools}

- DevTools → View Page Source
- DevTools → Network
- переход по якорной ссылке напрямую
