# Структурированные данные {#structured-data}

1. <strong>Уточняют смысл контента</strong> - поисковик четко определяет на странице важную информацию: цену товара, рейтинг, имя автора стати, дату публикации и т.д.
2. <strong>Формируют расширенные результаты поиска</strong> - снипет в органической выдаче имеет дополнительную информацию (rich results).
3. <strong>Повышаются CTR</strong> - расширенные сниппеты занимают больше места в выдаче, чаще привлекают внимание и обычно получают больше кликов.

## JSON-LD и Microdata

JSON-LD и Microdata это форматы/технологии размещения структурированных данных на странице.
Для описания типов и свойств (Article, Product, Organization и т.д.) используется словарь <code>Schema.org</code>.

<div class="text-block text-block--info mt-30">
  <p>Словарь Schema.org задаёт <i><strong>ЧТО</strong></i> представить поисковикам, а JSON-LD или Microdata - <i><strong>КАК</strong></i> это представить.</p>
</div>

JSON-LD и Microdata - предпочтительные форматы размещения структурированных данных на страницах сайта. Каждый со своими преимуществами и недостатками. Хорошо поддерживаются поисковыми системами Google и Яндекса.

| Критерий                      | JSON-LD          | Microdata      |
| ----------------------------- | ---------------- | -------------- |
| Влияние на HTML               | не меняет HTML   | засоряет HTML  |
| Поддержка JS-фреймворков      | отлично          | неудобно       |
| Поддержка Google              | предпочтительный | поддерживается |
| Поддержка Яндекса             | поддерживается   | поддерживается |
| Поддержка сложных структур    | проще            | сложнее        |
| Поддержка динамических данных | проще            | сложнее        |

На одной странице сайта можно использовать одновременно оба формата. Главное что бы данные не противоречили друг другу.

## Архитектурное отличие {#architectural-diff}

### JSON-LD {#jsonld-architecture}

<p>Данные описываются отдельным JSON-объектом.</p>

```HTML
<head>
  <script type="application/ld+json">
    {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "Title",
    "author": {
      "@type": "Person",
      "name": "John"
      }
    }
  </script>
</head>
```

Особенности:

- не связан напрямую с HTML
- находится в `<script>`
- описывает данные декларативно
- можно размещать в `<head>` без ущерба <a href="/checklists/technical/core-web-vitals.html">Core Web Vitals</a>
- JSON-LD можно просто сформировать из объекта:

```JS
schema = {
  "@type": "Product",
  name: product.name,
  price: product.price
}
```

<div class="text-block text-block--warning mt-30">
  <p>Недостаток JSON-LD в том, что возможна рассинхронизация данных с контентом, когда в HTML и JSON-LD разные значения.</p>
</div>

### Microdata {#microdata-architecture}

<p>Разметка встраивается прямо в HTML-элементы.</p>

```HTML
<article itemscope itemtype="https://schema.org/Article">
  <h1 itemprop="headline">Title</h1>
  <span itemprop="author">John</span>
</article>
```

Особенности:

- структура встроена в DOM
- привязана к конкретным элементам страницы

<div class="text-block text-block--warning mt-30">
  <p>Недостаток Microdata в том, что сильно загрязняется HTML. На больших страницах HTML становится трудночитаемым.</p>
</div>

## Что использовать {#what-to-use}

JSON-LD лучше для современных сайтов и SEO‑проектов, когда:

- контент генерируется сервером
- нужно масштабируемо и централизованно описывать структуру
- хочется не засорять HTML
- необходимо использовать сразу много типов Schema.org на странице (статьи, товары, отзывы и т.д.)

<br>

Microdata удобна для малых проектов, когда:

- сайт статический, без динамического JS
- разметка простая и небольшая (лендинги, корпоративные страницы, отдельные статьи)
- важно, чтобы данные были жёстко привязаны к HTML-контенту
- нет необходимости в сложной генерации или масштабировании

<br>

| Тип сайта            | Лучший вариант        |
| -------------------- | --------------------- |
| SSR                  | JSON-LD               |
| Большие сайты        | JSON-LD               |
| Маркетплейсы         | JSON-LD               |
| Новостные сайты      | JSON-LD               |
| Статические лендинги | Microdata или JSON-LD |
| Очень простой HTML   | Microdata             |
