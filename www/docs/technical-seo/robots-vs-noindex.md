# Robots vs Noindex - разница и применение {#robots-vs-noindex}

## Основные различия {#differences}

| Механизм     | Что делает                        | Уровень                 |
| ------------ | --------------------------------- | ----------------------- |
| `robots.txt` | Управляет **обходом (crawl)**     | До загрузки страницы    |
| `noindex`    | Управляет **индексацией (index)** | После загрузки страницы |

Если в двух словах:

- `robots.txt` - не пускать бота (если тебе важно ограничение)
- `noindex` - пустить, но не индексировать (если тебе важен контроль)

## Robots.txt {#robots}

Файл:

```
/robots.txt
```

Пример:

```
User-agent: *
Disallow: /admin/
```

### Как работает {#robots-how-it-works}

1. страница "заблокирована" для парсинга поисковиками
2. поисковик НЕ заходит на страницу
3. не видит HTML
4. не видит noindex, canonical, контент

### Когда использовать {#robots-when-to-use}

<ol>
  <li>Закрытие технических разделов:
    <ul>
      <li>/admin</li>
      <li>/api</li>
      <li>/cart</li>
      <li>/login</li>
    </ul>
  </li>
  <li>Ради экономии crawl-бюджета (для очень больших сайты)</li>
</ol>

### Ограничения и проблемы {#robots-problems}

- **Может не предотвратить индексацию** страницы или целой группы страниц. URL все равно могут попасть в индекс без контента, если на них ведут внутренние или внешние ссылки. Технических страниц типа `/cart` это не касается.
- **Не передаётся ссылочный вес** от страниц "заблокированных" в `Robots.txt`. Поисковой бот просто не увидит ссылки внутри "заблокированной" страницы.
- **Не работает с canonical и meta robots**

## Noindex {#noindex}

Meta-тег:

```HTML
<meta name="robots" content="noindex, follow">
```

или HTTP header:

```
X-Robots-Tag: noindex
```

### Как работает {#noindex-how-it-works}

1. страница "НЕ заблокирована" для парсинга поисковиками
2. поисковик заходит на страницу и анализирует её
3. учитывается контент, перелинковка, canonical, мета-теги
4. страница **НЕ добавляется в индекс**

<div class="text-block text-block--info mt-30">
  <p>Даже с noindex страница помогает SEO всего сайта, потому что поисковик понимает её структуру и связи.</p>
</div>

### Когда использовать {#noindex-when-to-use}

<ol>
  <li>Есть дубли:
    <ul>
      <li>фильтры (?color=red)</li>
      <li>сортировки (?sort=price)</li>
    </ul>
  </li>
  <li>Есть пагинация (иногда)</li>
  <li>Есть служебные страницы:
    <ul>
      <li>поиск по сайту</li>
      <li>временные страницы</li>
    </ul>
  </li>
</ol>

### Варианты исполнения {#variants}

```JS
noindex, follow    // стандарт
noindex, nofollow  // редко

```

Значение `follow` используется чтоб передавался **ссылочный вес** и работала **перелинковка**.

### Правила для Noindex {#noindex-rules}

<ul>
  <li> 
    <p><strong>Правило №1</strong></p>
    <p>На странице, закрытой в robots.txt, использовать <code>noindex</code> бессмысленно. По причине того, что бот не зайдёт на эту страницу и не увидит её содержимое.</p></li>

  <li>
    <p><strong>Правило №2</strong></p>
    <p>Удаление страницы из индекса выполняется через <code>noindex</code>, а не через <code>Disallow</code>.</p>
    Как выполнить:
    <ol>
      <li>убрать Disallow (если есть)</li>
      <li>поставить noindex</li>
      <li>дождаться удаления</li>
      <li>снова закрыть через Disallow (опционально)</li>
    </ol>
  </li>
  <li>
    <p><strong>Правило №3</strong></p>
    <p>Robots не гарантирует защиту от индексации - это только управление обходом.</p>
  </li>
</ul>

## Robots vs Noindex {#vs}

<strong>robots.txt</strong>, если:

- страница не имеет ценности вообще
- не нужна индексация
- не нужна передача веса

<strong>noindex</strong>, если:

- важно сохранить SEO-структуру
- страница нужна пользователю
- но не нужна в поиске

## Типичные ошибки {#mistakes}

<ol>
  <li> Неправильное удаление из индекса. Мусор в индексе все равно останется, и может даже множиться.

```
Disallow: /filter/
```

  </li>
  <li> Так <code>noindex</code> не работает.

```
noindex + Disallow
```

  </li>
  <li>Индексируемые фильтры без контроля →  каннибализация и создание дублей.</li>
</ol>

## Рекомендации {#guide}

<ol>
  <li>Canonical, который используется для склейки дублей и передачи веса, требует доступности страницы. Тут нужно использовать <code>noindex</code>, а не <code>robots</code>.</li>
  <li>Sitemap должен содержать только индексируемые страницы, и не должен включать <code>noindex</code>.</li>
  <li>Для фильтров (не имеющих ценности) и сортировок лучше использовать <code>noindex, follow</code> и <code>canonical</code> на основную категорию или ближайшую релевантную SEO-важную страницу.
  </li>
  <li>Для пагинации правильно использовать self-canonical. Для глубоких страниц пагинации можно использовать <code>noindex</code>.</li>
  <li>Для технических страниц правильно использовать <code>Disallow</code> в <code>robots.txt</code>.</li>
</ol>
